Installation

md-utils currently supports macOS 13 and newer. Version 0.1.0 is source-build only, so installing through Mint or Swift Package Manager requires a Swift 6-capable build environment, usually Xcode 16+.

Mint 
mint install DandyLyons/md-utils@0.1.0
mint run DandyLyons/md-utils@0.1.0 --help

Mint builds the Swift executable package from the tagged GitHub source.

From Source 
git clone https://github.com/DandyLyons/md-utils.git
cd md-utils
swift build -c release

Swift Package Manager builds the MarkdownUtilities library and the md-utils executable from source.

Agent Skill 
npx skills add https://github.com/DandyLyons/md-utils --skill markdown-utilities

The bundled skill teaches AI coding assistants how to use md-utils for structured Markdown operations.

Quick Start


md-utils toc README.md
md-utils fm dump document.md
md-utils extract --name "Installation" README.md
md-utils links check --vault ~/notes/ docs/
md-utils schema init books --path "Books/**/*.md" --tag Book

Hosted Schema

The md-utils configuration schema is published with this site so editors and tools can reference a stable public URL. The current compatibility endpoint is https://dandylyons.github.io/md-utils/md-utils.schema.json.

Versioned schema files are available under /schemas/0.1.0/md-utils.schema.json, with /schemas/latest/md-utils.schema.json maintained as a moving alias for the latest published schema.

{
  "$schema": "https://dandylyons.github.io/md-utils/md-utils.schema.json",
  "schemaDirectory": ".md-utils/schemas/",
  "schemaRules": []
}
